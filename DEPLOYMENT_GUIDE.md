# 幼儿园作业进度追踪系统 - 部署指南

## 静态网站部署

要将应用部署为公开可访问的网站，您可以选择以下任一平台：

### 方案一：GitHub Pages
1. 注册并登录代码托管平台
2. 创建新仓库并上传 `index.html` 文件
3. 在仓库设置中启用 Pages 功能
4. 几分钟后，您将获得一个公开的网站链接

### 方案二：Vercel
1. 访问 Vercel 官网并注册账号
2. 创建新项目并上传您的项目文件
3. 一键部署，获得公开链接

## 数据同步解决方案

当前应用使用浏览器本地存储，数据仅保存在单个设备上。要实现多设备数据同步，需要使用云数据库服务。

### 推荐方案

1. **选择云服务提供商**：注册一个提供实时数据库服务的云平台账号。

2. **创建数据库实例**：
   - 在云平台控制台中创建一个新的实时数据库项目
   - 记录下自动生成的配置信息

3. **修改应用代码**：
   - 在 HTML 文件头部引入相应的 JavaScript SDK
   - 使用您的配置信息初始化云服务
   - 将原有的本地存储读写函数替换为云数据库的 API 调用

4. **重新部署**：
   - 将修改后的代码重新部署到静态网站托管平台

## 数据有效期设置

要实现7天数据自动过期功能，您可以：
- 在数据库中存储一个时间戳字段
- 应用启动时检查当前时间与存储时间的差值
- 如果差值超过7天，则清空数据并重新初始化

## 注意事项

- 请确保您了解并遵守所选云服务的使用条款和免费额度限制
- 对于图片等大型资源，建议使用专门的云存储服务
- 定期备份您的重要数据

## 后续优化建议

- 为应用添加简单的访问控制，确保数据安全
- 优化图片上传功能，自动压缩图片以节省存储空间
- 根据实际使用情况，考虑添加数据导出和备份功能
